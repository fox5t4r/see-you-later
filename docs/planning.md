## 핵심 의사결정 기록

### 결정 1: 왜 크롬 확장 프로그램인가?

| 대안 | 장점 | 단점 | 결정 |
|------|------|------|------|
| 웹 앱 | 설치 불필요 | 현재 보고 있는 페이지 접근 불가 | ❌ |
| 데스크톱 앱 | 강력한 기능 | 배포 복잡, 크로스 플랫폼 이슈 | ❌ |
| **크롬 확장** | **현재 탭 콘텐츠 직접 접근 가능** | 크롬 전용 | ✅ |

**결정 근거**: "지금 보고 있는 페이지를 바로 요약"하려면 브라우저 확장이 유일한 선택지. 사용자가 URL을 복사해서 붙여넣는 단계를 제거하는 것이 핵심 UX.

### 결정 2: 왜 Gemini인가? (v0.2.0 전환)

```
v0.1.0의 문제점:
┌─────────────────────────────────────────────┐
│  사용자가 해야 할 것:                         │
│  1. Docker Desktop 설치                      │
│  2. docker-compose up 실행                   │
│  3. Claude API 키 발급 (유료)                │
│  4. OpenAI API 키 발급 (유료)                │
│  5. 크롬 확장 설치                            │
│                                              │
│  → 비전공자에게 사실상 사용 불가능              │
└─────────────────────────────────────────────┘

v0.2.0의 해결:
┌─────────────────────────────────────────────┐
│  사용자가 해야 할 것:                         │
│  1. ZIP 다운로드 & 압축 해제                  │
│  2. 크롬에서 확장 로드                        │
│  3. Gemini API 키 발급 (무료)                │
│                                              │
│  → 누구나 3분 안에 설치 가능                  │
└─────────────────────────────────────────────┘
```

| 비교 항목 | Claude + Whisper (v0.1.0) | Gemini (v0.2.0) |
|-----------|--------------------------|-----------------|
| 사용자 비용 | 유료 (Claude + OpenAI) | **무료** (Gemini 무료 티어) |
| 설치 난이도 | Docker 필요 | **ZIP 해제만** |
| 자막 없는 영상 | Whisper 음성 변환 | **Gemini 멀티모달** (URL 직접 전달) |
| 서버 운영 | FastAPI 서버 필요 | **서버 불필요** |
| 요약 품질 | Haiku 4.5 (빠르지만 얕음) | **2.5 Pro** (깊이 있는 분석) |

### 결정 3: SDK vs REST API 직접 호출

```
문제 상황:
  @google/generative-ai  → deprecated (2025년 말)
  @google/genai           → Node.js 전용 (Chrome Service Worker 호환 불확실)

선택지:
  A) @google/genai SDK 사용 → 서비스 워커에서 동작 안 할 위험
  B) REST API 직접 호출    → fetch()만 있으면 어디서든 동작

결정: B (REST API 직접 호출)
  → 브라우저 환경에서 100% 동작 보장
  → SDK 업데이트/deprecation에 영향받지 않음
  → 요청/응답 구조를 완전히 제어 가능
```

### 결정 4: YouTube 자막 추출 — 하이브리드 전략

```
YouTube 영상 요약 흐름:

  영상 URL 입력
       │
       ▼
  ┌─────────────────┐
  │ 자막 있는가?      │
  └────┬────────┬───┘
       │        │
      Yes       No
       │        │
       ▼        ▼
  ┌─────────┐  ┌──────────────────┐
  │ 텍스트   │  │ Gemini 멀티모달    │
  │ 기반 요약 │  │ (YouTube URL을    │
  │          │  │  fileData로 전달)  │
  └─────────┘  └──────────────────┘
       │        │
       ▼        ▼
  토큰 절약    비용 높지만
  (텍스트만)   자막 없어도 처리 가능
```

**왜 하이브리드인가?**
- 자막이 있는 영상 (전체의 ~80%): 텍스트만 보내면 토큰 비용 1/10
- 자막이 없는 영상 (전체의 ~20%): 멀티모달로 영상 자체를 분석
- 무조건 멀티모달로 보내면 무료 한도를 빠르게 소진

### 결정 5: Watch Later 자동 요약 — 어떻게 목록을 가져오는가?

```
시도 1: YouTube Data API v3
  → 문제: OAuth 2.0 필요, 사용자에게 Google 로그인 동의 화면 표시
  → 크롬 확장에서 OAuth 플로우가 복잡하고 사용자 이탈 위험

시도 2: YouTube 페이지 DOM 파싱
  → 문제: YouTube는 SPA라서 DOM이 동적으로 변경
  → 페이지 로드 타이밍에 따라 추출 실패

시도 3 (최종): ytInitialData 글로벌 변수 파싱
  → YouTube 페이지의 초기 데이터가 window.ytInitialData에 JSON으로 존재
  → chrome.scripting.executeScript로 주입하여 추출
  → DOM 변경에 영향받지 않아 안정적
```
